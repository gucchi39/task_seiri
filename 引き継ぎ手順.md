# Claude Code への引き継ぎ手順

## 1. ファイルを手元に用意

このチャットから次のファイルをダウンロードして、ローカルの作業フォルダにまとめる。

- `index.html`（ミニアプリ／localStorage保存版）
- `README.md`
- `CLAUDE.md`（Claude Code が自動で読む文脈ファイル）

任意で `docs/` フォルダを作り、以下も入れておくと成果物置き場になる：
- `タグチ刺繍ネーム_業務分析レポート.md`
- `タグチ刺繍ネーム_業務分析.xlsx`
- `工場長_業務棚卸し.xlsx`

## 2. リポジトリをクローン

```bash
git clone https://github.com/gucchi39/task_seiri.git
cd task_seiri
```

（まだ空のリポジトリ。ここに上のファイルをコピーして入れる）

## 3. Claude Code を起動

`task_seiri` フォルダの中で Claude Code を起動する。
`CLAUDE.md` が同じフォルダにあるので、文脈は自動で読み込まれる。

最初にこう伝えればOK：

```
このフォルダの index.html（業務棚卸しマトリックスのミニアプリ）を
コミットして GitHub に push して。そのあと GitHub Pages を有効化する
手順を教えて。公開URLは https://gucchi39.github.io/task_seiri/ の想定。
詳しい背景は CLAUDE.md を見て。
```

## 4. push まで（Claude Code がやってくれるコマンド）

```bash
git add .
git commit -m "Add task organization matrix mini-app"
git branch -M main
git push -u origin main
```

## 5. GitHub Pages を有効化

GitHub のリポジトリ画面で：
Settings → Pages → Build and deployment → Source を **Deploy from a branch**、
Branch を **main / (root)** にして Save。

数分待つと以下で公開される：
https://gucchi39.github.io/task_seiri/

工場長にはこのリンクを渡すだけで使ってもらえる。

---

## メモ

- 以後の修正（軸ラベル・色・機能追加）は Claude Code に日本語で指示すればそのまま反映＆pushできる。
- 保存は各利用者のブラウザ内（localStorage）。面談で工場長が置いた配置は、その端末では残るが他端末には共有されない。共有したい場合は別途相談を。
